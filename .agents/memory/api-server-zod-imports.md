---
name: API server zod imports
description: How API routes should validate input given the API server package does not list zod as a direct dependency.
---

API routes must validate request bodies and query parameters using schemas generated by `@workspace/api-zod`.

**Why:** The `@workspace/api-server` package does not list `zod` as a direct dependency. `zod` is available transitively through `@workspace/api-zod`, but pnpm does not hoist it for direct import in the API server. Importing `zod` directly causes `ERR_MODULE_NOT_FOUND` at runtime and `TS2307` at type-check time.

**How to apply:**
- Use the generated schemas exported by `@workspace/api-zod` (e.g., `RegisterBody`, `LoginBody`, `CreateLfgPostBody`, `RedeemActivationCodeBody`, `AdminActivateProBody`, `CreateActivationCodeBody`).
- Parse request input with these schemas in the route handler, then pass the typed result to the underlying service/helper functions.
- If a new endpoint needs validation that does not yet exist in `@workspace/api-zod`, add the input/body schema to `lib/api-spec/openapi.yaml`, run `pnpm --filter @workspace/api-spec run codegen`, and import the generated schema in the route.
